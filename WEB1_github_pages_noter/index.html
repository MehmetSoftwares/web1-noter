Spørgsmål 1 – Projekt 1: Responsivt design (Flexbox, Grid, media queries)
  • Projekt 1 er et CV-website bygget responsivt.
  • Overordnet layout laves med CSS Grid (2D layout).
  • Komponenter (nav, små sektioner) laves med Flexbox (1D layout).
  • Media queries skifter layout ved breakpoints (kolonner → én kolonne på mobil).
  • Billeder skalerer med max-width: 100% og height: auto.
Åbn i IDE (hurtigst): index.html + style.css (eller tilsvarende).
Eksempel (CSS) 
/* Grid til side-layout */
.container {
  display: grid;
  grid-template-columns: 1fr 2fr;
  gap: 2rem;
}

/* Flexbox til navigation */
nav ul {
  display: flex;
  justify-content: space-between;
}

/* Media query til mobil */
@media (max-width: 768px) {
  .container { grid-template-columns: 1fr; }
  nav ul { flex-direction: column; }
}

/* Fleksible billeder */
img { max-width: 100%; height: auto; }
Spørgsmål 2 – CSS box model og box-sizing: border-box
  • Box model: content → padding → border → margin.
  • Default content-box: width/height gælder kun content (padding+border lægges oveni).
  • border-box: width/height inkluderer padding+border → mere forudsigeligt layout.
Eksempel 
/* Uden border-box (standard content-box) */
.element {
  width: 200px;
  padding: 20px;
  border: 5px solid;
}
/* Total bredde = 200 + 2*20 + 2*5 = 250px */

/* Med border-box */
* { box-sizing: border-box; }
.element {
  width: 200px;      /* inkluderer padding + border */
  padding: 20px;
  border: 5px solid;
}
/* Total bredde = 200px */
Spørgsmål 3 – Strategier for bedre webtilgængelighed (accessibility)
  • Semantisk HTML (<header>, <nav>, <main>, headings i orden).
  • alt-tekster på billeder.
  • Tastatur: tab-rækkefølge, synlig fokus, ingen “keyboard traps”.
  • Kontrast + ikke kun farve som information.
  • Labels på forms + ARIA kun når nødvendigt.
  • Test: Lighthouse + manuel keyboard/zoom.
Mini-eksempler 
<img src="profil.jpg" alt="Portrætbillede af Mehmet">

<label for="email">Email</label>
<input id="email" type="email">

button:focus {
  outline: 3px solid black;
}
Spørgsmål 4 – DOM: hvad er det, og hvordan interagerer JavaScript?
  • DOM er browserens træ-repræsentation af HTML (noder/elementer).
  • JS kan finde elementer, ændre tekst/klasser/styles og reagere på events.
  • Værdifuld demo: selection + event + opdatering af DOM.
Live-demo 
<h1 id="title">Hej</h1>
<button id="btn">Klik</button>

<script>
const title = document.getElementById("title");
const btn = document.getElementById("btn");

btn.addEventListener("click", () => {
  title.textContent = "Hej verden!";
});
</script>

Spørgsmål 3 – UI vs UX
  • UI = det visuelle: knapper, farver, typografi, layout.
  • UX = oplevelsen: flow, feedback, effektivitet, frustration vs. tilfredshed.
  • Du kan have flot UI og dårlig UX – og omvendt, men god UX kræver ofte fornuftig UI.
Eksempel: UI = “Send”-knappens look. UX = hvad der sker når man trykker (feedback, fejl, bekræftelse).

Spørgsmål 2 – Media queries og breakpoints
  • Media query = betinget CSS (fx baseret på width).
  • Breakpoint = punkt hvor layout ændrer sig.
  • Typisk: kolonner → stak på mobil.
Eksempel 
.container { display: flex; flex-direction: row; }

@media (max-width: 768px) {
  .container { flex-direction: column; }
}
Spørgsmål 3 – HTML-struktur og semantisk HTML
  • Grundstruktur: <head> (meta/links) + <body> (indhold).
  • Semantik: brug <header>, <nav>, <main>, <section>, <footer>.
  • Fordele: accessibility, SEO, vedligeholdelse.
Skeleton 
<!DOCTYPE html>
<html lang="da">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Titel</title>
</head>
<body>
  <header>...</header>
  <main>...</main>
  <footer>...</footer>
</body>
</html>
Spørgsmål 4 – Fordele/ulemper ved JavaScripts dynamiske typning
  • Fordele: hurtigt at skrive, fleksibelt, godt til prototyping.
  • Ulemper: type-fejl opdages ved runtime, implicit coercion kan give bugs.
  • Best practice: brug ===, vær eksplicit, evt. TypeScript i større projekter.
Eksempler 
let x = 42;
x = "hej";     // tilladt (dynamisk)

"5" + 2;       // "52" (string concat)
"5" - 2;       // 3 (coercion til number)

0 == false;    // true
0 === false;   // false

Spørgsmål 4 – JSON: struktur + serialize/deserialize
  • JSON er et dataformat (tekst) til udveksling.
  • JS bruger JSON.stringify og JSON.parse.
  • JSON kan ikke indeholde funktioner – kun data.
Eksempel

Spørgsmål 4 – Fejlhåndtering ved fetch
  • Fetch kaster typisk kun fejl ved netværksfejl – ikke automatisk ved 404/500.
  • Derfor: tjek response.ok og kast fejl selv.
Robust fetch

  .then(response => {
    if (!response.ok) throw new Error("HTTP-fejl");
    return response.json();
  })
  .then(data => console.log(data))
  .catch(err => console.error("Noget gik galt:", err));

Spørgsmål 2 – Absolutte og relative sizing units (CSS)
  • Absolutte: px, cm (faste).

Spørgsmål 3 – Hvordan form submissions fungerer
  • Submit sender data baseret på name.
  • action = destination, method = GET/POST.
  • JS kan stoppe reload med preventDefault().
HTML + JS 
<form id="f" action="/submit" method="post">
  <input name="email" type="email">
  <button type="submit">Send</button>
</form>

<script>
const form = document.getElementById("f");
form.addEventListener("submit", (e) => {
  e.preventDefault(); // stop reload
  // her kan du sende med fetch
});
</script>
